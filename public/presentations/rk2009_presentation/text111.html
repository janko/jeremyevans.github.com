Notes:

I release a new version of Sequel every month near the beginning of the month.

Releases have a history of being very stable, and Sequel can easily stick to a time based release cycle because at all times the master branch is in a releasable state.

This is due to the fact that before pushing any code modifications to github, I run the full release test suite.

A consequence of this is that the master branch is generally the most bug-free version of Sequel.
